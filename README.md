# Railway Transportation System 🧑‍💻
# پیاده‌سازی سیستم شرکت حمل و نقل ریلی 🚅
این پروژه یک نرم‌افزار خط فرمان (CLI) برای مدیریت خطوط و قطارها، خرید بلیط توسط کاربران و مدیریت کارکنان است. این سیستم با سه نقش اصلی پیاده‌سازی شده:
مدیر (Admin)، کارمند قطار (Train Employee) و کاربر عادی (User)
# ویژگی‌ها 💠
•	مدیر: ورود با نام کاربری و رمز عبور ثابت، مدیریت کارمندان (افزودن، حذف، مشاهده لیست)

•	کارمند: مدیریت خطوط (افزودن، ویرایش، حذف، نمایش) و مدیریت قطارها (افزودن، ویرایش، حذف، نمایش) با اختصاص شناسه یکتا به هر قطار

•	کاربر: ثبت‌نام، ورود، شارژ کیف پول (با اعتبارسنجی کارت بانکی بر اساس قالب)، خرید بلیط، مشاهده موجودی و تراکنش‌ها، ویرایش اطلاعات شخصی

•	تولید خودکار فایل بلیط (متن) پس از خرید

•	ذخیره اطلاعات کیف پول و تراکنش‌ها در فایل

•	اعتبارسنجی ورودی‌ها (ایمیل، رمز عبور، نام کاربری، شماره کارت و ...)


# پیش‌نیازها ↪️
•	پایتون 3.6 یا بالاتر

•	هیچ کتابخانه خارجی خاصی نیاز نیست (فقط ماژول‌های داخلی پایتون)




# نحوه نصب و اجرا 📝
1- فایل‌های پروژه را در یک پوشه قرار دهید همه فایل‌های .py

2- ترمینال را در آن پوشه باز کنید.

3- دستور زیر را اجرا کنید:    python main.py

4- از منوی اصلی گزینه مورد نظر را انتخاب کنید.



# ساختار پروژه 🖥️

mangagement_panel.py:

add_employee.py:

delete_employee.py:	

حذف کارمند با نام کاربری

list_employee.py:

نمایش لیست کارمندان

employee_menu.py:	

ورود کارمند و فراخوانی پنل کارمند

employee_panel.py:

پنل کارمند (مدیریت خطوط و قطارها)

Train_employee.py:

BANK.py:

# راهنمای استفاده  ❇️
منوی اصلی :

پس از اجرا، گزینه‌های زیر نمایش داده می‌شود:
1.	-Admin  ورود به پنل مدیریت
2.	Train Employee  -ورود کارمند
3.	User -ورود/ثبت‌نام کاربر
4.	Exit -خروج از برنامه

نقش مدیر (Admin) :

•	نام کاربری پیش‌فرض grp5

•	رمز عبور پیش‌فرض 123456

•	پس از ورود موفق، پنل مدیریت کارمندان با گزینه‌های:

o	افزودن کارمند (با رعایت قوانین اعتبارسنجی)

o	حذف کارمند (بر اساس نام کاربری)

o	نمایش لیست کارمندان

نقش کارمند (Train Employee) :

•	ورود با نام کاربری و رمز عبور کارمندانی که توسط مدیر اضافه شده‌اند.

•	پنل کارمند شامل :

o	مدیریت خطوط: افزودن خط جدید (با نام، مبدأ، مقصد، تعداد ایستگاه‌ها و لیست ایستگاه‌ها)، ویرایش خط، حذف خط، نمایش لیست خطوط

o	مدیریت قطارها: افزودن قطار به یک خط موجود (با نام، سرعت، زمان توقف، قیمت، ظرفیت، کیفیت)، ویرایش قطار، حذف قطار، نمایش لیست قطارها

o	هر قطار دارای شناسه یکتا مثلاً (TN-1) است.

نقش کاربر (User) :

•	ثبت‌نام: نام، نام کاربری (یکتا)، رمز عبور (حداقل یک حرف، یک عدد و یکی از کاراکترهای @ یا &)، ایمیل معتبر و یکتا

•	ورود: با نام کاربری و رمز عبور ثبت‌شده

•	پس از ورود موفق، کاربر می‌تواند به پنل خرید برود:

o	خرید بلیط: نمایش قطارهای موجود (با ظرفیت باقی‌مانده)، انتخاب قطار و تعداد بلیط. در صورت ناکافی بودن موجودی کیف پول، امکان شارژ وجود دارد.

o	شارژ کیف پول: وارد کردن اطلاعات کارت بانکی (بر اساس قالب‌های تعریف شده در BANK.py). پس از تأیید، موجودی افزایش می‌یابد.

o	ویرایش اطلاعات: تغییر نام، ایمیل یا رمز عبور (با رعایت قوانین)

o	مشاهده کیف پول: نمایش موجودی و تاریخچه تراکنش‌ها، ذخیره در فایل wallet_<username>.txt

o	خروج از پنل خرید

پس از خرید موفق، یک فایل بلیط با نام ticket_<username>_<timestamp>.txt در همان پوشه ساخته می‌شود.



# نکات مهم ⚠️

اطلاعات پیش‌فرض مدیر :

•	نام کاربری grp5

•	رمز عبور 123456

اعتبارسنجی کارت بانکی (برای شارژ کیف پول) :

تابع validate_card در فایل BANK.py کارت را بر اساس قالب زیر تأیید می‌کند :

•	شماره کارت: دقیقاً ۱۶ رقم (فقط اعداد)

• سه یا چهاررقمی بودن CVV2

•	تاریخ انقضا: فرمت MM/YY مثلاً 12/25 و تاریخ نباید گذشته باشد.

•	رمز کارت: حداقل ۴ رقم


قوانین اعتبارسنجی :

•	نام کاربری (کارمند): فقط حروف، حداقل ۳ کاراکتر

•	رمز عبور (کارمند): دقیقاً ۴ کاراکتر شامل حرف و عدد مثلاً a1b2

•	رمز عبور (کاربر عادی): حداقل یک حرف، یک عدد و یکی از @ یا & مثلاً a1@b

•	ایمیل: از یک الگوی استاندارد پیروی می‌کند بررسی با regex


ذخیره‌سازی :

•	اطلاعات کارمندان، کاربران، خطوط و قطارها فقط در حافظه (RAM) ذخیره می‌شود و پس از خروج از برنامه از بین می‌رود.

•	فایل‌های بلیط و گزارش کیف پول روی دیسک ذخیره می‌شوند.
